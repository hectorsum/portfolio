# Internationalization (i18n) Guide

The portfolio now supports English and Spanish translations. Language switching is handled via i18next and react-i18next.

## How to Use

A language switcher button appears in the header showing the current language flag (🇺🇸 for English, 🇪🇸 for Spanish). Click to change languages.

Language preference is persisted in localStorage, so the user's choice is remembered across sessions.

## File Structure

```
src/
  i18n.js                       i18n configuration and initialization
  locales/
    en.json                     English translations
    es.json                     Spanish translations
  components/
    LanguageSwitcher.jsx        Language toggle component (in Header)
```

## Adding New Translations

### 1. Add to translation files

In `src/locales/en.json`:
```json
{
  "mySection": {
    "title": "My Title",
    "description": "My description"
  }
}
```

In `src/locales/es.json`:
```json
{
  "mySection": {
    "title": "Mi Título",
    "description": "Mi descripción"
  }
}
```

### 2. Use in components

```jsx
import { useTranslation } from 'react-i18next';

export const MyComponent = () => {
  const { t } = useTranslation();
  
  return (
    <div>
      <h1>{t('mySection.title')}</h1>
      <p>{t('mySection.description')}</p>
    </div>
  );
};
```

## Translation Keys

All content is organized hierarchically:

- `nav.*` — Navigation links
- `hero.*` — Home page hero section
- `projects.*` — Projects page
- `about.*` — About page
- `expertise.*` — Expertise/experience page
- `contact.*` — Contact page
- `footer.*` — Footer

## Current Coverage

✓ All navigation links
✓ Hero section (title, CTA, stats)
✓ Projects page (title, descriptions, filters)
✓ About section (bio, skills, info card)
✓ Contact form (labels, placeholders, success message)
✓ Expertise timeline
✓ Footer navigation and copyright

## Switching Languages Programmatically

```jsx
const { i18n } = useTranslation();

// Change to Spanish
i18n.changeLanguage('es');

// Change to English
i18n.changeLanguage('en');

// Get current language
console.log(i18n.language); // 'en' or 'es'
```

## Supported Languages

- **en** — English
- **es** — Español (Spanish)

To add more languages:
1. Create `src/locales/xx.json` (where xx is the language code)
2. Add translations for all keys
3. Update `src/i18n.js` resources
4. Update `LanguageSwitcher.jsx` to include the new language button

## Storage

Language preference is stored in localStorage as `language`. Clear it to reset to browser default (English).
